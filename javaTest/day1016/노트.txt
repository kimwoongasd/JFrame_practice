# 메소드 호출 방식
- 값에 의한 호출(Call By Value)
	- 메소드 호출시에 기본자료형이 전달되는 것을 말한다
	- 메소드 호출시에 기본자료형이 매개변수로 전달될때에 그 변수의 값이 복사되어서 전달되어서
	  메소드안에 값을 변경할떄에 메소드 호출한 쪽에는 변경된 내용이 적용되지 않는다
	
- 참조에 의한 호출(Call By Reference)
	- 메소드 호출시에 참조자료형(배열, 클래스 객체)이 전달되는 것을 말한다
	- 배열이나 클래스의 객체 변수는 그 변수 자신이 값을 갖고있는 것이 아니라 값이 있는 메모리를 참조
	- 메소드 호출시에 배열(객체변수) 전달하면 값이 전달되는 것이 아니라 주소가 전달되기 떄문에
	  메소드 안에서 내용이 변경되면 변경된 내용이 적용이 된다
	  
	  
# 정렬
값을 순서대로 배치하는 것

## 오름차순 정렬
- 값이 작은 것 부터 순서대로 배치하는 것

## 내림차순 정렬
- 값이 큰 것 부터 순서대로 배치하는 것


# 클래스
- 새로운 자료형(Data Type)
- 기본 자료형만으로는 프로그램 하기가 복잡

# 객체 지향 프로그래밍 (Object Oriented Programming)
- 현실세계의 모든 것을 "객체화" 할 수 있다

# 객체
- 이세상을 구성하고 있는 모든 것은 객체이다.
- 눈에 보이는 것일수도 있고 보이지 않는 개념적인 것일 수 있다

- 각 객체는 그 객체를 구성하기 위한 여러가지 "속성"과 "동작"들로 구성되어있다

자바에서는 객체를 표현하기 위해서는 그 객체를 구성하고 있는
속성과 동작을 분석하여 클래스를 만든다
속성 : 맴버변수(클래스를 구성하고 있는)
동작 : 맴버메소드(클래스를 구성하고 있는)

# 객체 지향 프로그래밍의 특징
1. 은닉성(Encapsulation)
- 클래스를 구성하고 있는 속성(맴버변수)들은 외부로 노출하지 않고 보호하는 것
## 접근 명시자

- 동작(맴버메소드)들은 public 영역에 두오 외부로부터 접근을 허용
private		: 외부에 노출하지 않을 맴버
prulbic		: 외부에 노출할 맴버
protected
